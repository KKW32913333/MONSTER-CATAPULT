# 🎯 MONSTER CATAPULT（モンスター・カタパルト）

添付いただいたゲームストアイメージ／画面サンプルを参考に、一から再構築したバージョンです。
ワールドマップ形式のステージ選択、モンスター育成ショップ、デイリーミッション、オンラインランキングを備えた、
より製品に近い構成になっています。

**デザイン**：太いアウトライン＋フラットな塗り＋オフセット影の「ステッカー調」スタイル（Clash Royale/Brawl Starsなどに近い、明るく親しみやすいモバイルゲームらしい配色）。フォントは丸ゴシックの「M PLUS Rounded 1c」（Google Fonts、CDN経由で読み込み）。

---

## 🎮 ゲームの遊び方

1. ワールドマップでステージを選択（クリア済みステージの隣が次に挑戦可能）
2. 画面左下の砲台をドラッグして狙いを定め、指を離して発射
3. モンスターごとの特殊効果を活かして砦を崩し、敵を全滅させればクリア
4. 弾を多く残してクリアするほど獲得★（1〜3）が増え、報酬ゴールドも増加
5. ゴールドでモンスターを解放・強化し、次のステージに挑もう

## 👾 モンスター一覧（6種）

| モンスター | 特殊効果 | 入手方法 |
|---|---|---|
| 🟢 スライム | 着弾で3体に分裂し、周囲へ被害を広げる | 最初から所持 |
| 🐉 ファイアドラゴン | 着弾で爆発し、範囲内の敵を一掃する | 最初から所持 |
| ❄️ アイスゴーレム | 着弾で周囲の敵を凍結させ、動きを止める | 最初から所持 |
| 🟣 スパイクボール | 弾みやすく、何度も跳ねて多段ヒットしやすい | 最初から所持 |
| 💀 スケルトンナイト | 質量が大きく、構造物を力任せに突き崩す | ショップで解放（600G） |
| 🏹 ウィンドケンタウルス | 飛行中に浮力がかかり、伸びのある低伸弾道になる | ショップで解放（900G） |

各モンスターは最大Lv5まで強化可能。レベルが上がると「パワー」が上昇し、より低い衝撃速度で敵を撃破できるようになります。

## ✨ 主な機能・システム

- **🗺️ ワールドマップ形式のステージ選択**：全10ステージ、5の倍数はBOSSステージ表示。星評価つき
- **🎯 ドラッグ＆リリース操作**：Matter.jsによるリアルな物理演算でモンスターを発射
- **💥 モンスター特殊効果**：分裂・爆発・凍結・跳躍・貫通・重量など個性豊かな着弾エフェクト
- **📦 NEXTキュー（個数表示）**：残りの弾をモンスター種別ごとにまとめて表示
- **🛡️ 耐久力のある敵（後半ステージ）**：ステージ7以降は敵が2ヒット制になり、瞬間火力だけでなく立ち回りが問われる
- **🏰 城のシルエット構造**：単なるブロックの山積みではなく、両脇の塔＋城壁＋胸壁（凹凸）で構成された「城」の形に再設計。ボスステージは中央にひときわ高い主塔（キープ）が加わる。塔の頂上には旗が立ち、塔が崩れると旗も一緒に傾く
- **👑 ボスステージ（5・10）**：巨大なボス敵が登場。複数回の攻撃が必要な高耐久・高得点の特別な敵。専用の演出（紫のオーラ・大型HPバー）付き
- **🛢️ 爆発樽**：ステージ3以降、砦の脇に設置。当てると誘爆し周囲を巻き込む。樽同士が近いと連鎖爆発する
- **🛒 ショップ／モンスター図鑑**：モンスターの解放・強化、パワー／重さのステータス確認
- **📋 デイリーミッション（3種同時）**：日替わりで3つのミッションが提示され、達成でゴールド獲得
- **🖼️ ステージ背景イラスト**：ステージ進行に合わせて背景が切り替わる（緑の草原→森の小川→水晶の洞窟→雪山の村）。ボスステージ（5・10）は専用の溶岩渓谷背景に
- **📱 PWA対応**：ホーム画面に追加してアプリのように起動可能。オフライン動作、新バージョン公開時は自動リロードで最新化
- **🎨 アプリアイコン**：専用デザインのアイコン画像を使用（`icon-58.png` 〜 `icon-1024.png` をルート直下に同梱）
- **🏆 オンラインランキング**：Firebase連携で合計スコアを世界中の冒険者と競える（任意設定）
- **💾 端末保存**：ゴールド・モンスター所持状況・ステージ進捗・ミッション進捗をlocalStorageに保存
- **🎬 画面演出**：着弾・撃破時の画面シェイク、破片パーティクル、モンスターの飛翔軌跡を追加

## 🖼️ 画質改善（高解像度対応）

以前はCanvas（ゲーム画面本体）が高DPI端末（iPhone等のRetinaディスプレイ）に対応しておらず、
画面全体がぼやけて表示される不具合がありました。Canvasの内部描画解像度をdevicePixelRatioに
合わせて拡大するよう修正し、画面全体がくっきり表示されるようになりました。

あわせて、ステージ背景・砦のブロック・モンスター画像も、元のアップロード画像を活かしてより
高い解像度で書き出し直しています（背景: 1170×1380、ブロック: 400×400、モンスター: 480×480）。

## 🐞 物理演算の不具合修正

以前のバージョンでは、砦の中段に配置する敵をブロックとブロックの隙間に埋め込む形で配置していたため、
ステージ開始時に物理演算が重なりを解消しようとして敵やブロックが画面外へ吹き飛ぶ、
または隙間に挟まって直接攻撃が届かなくなる、という不具合がありました。

現在は全ての敵を「ブロックの上に露出したステップ（棚）」に配置する方式に変更し、
初期状態での重なりが発生しないよう修正しています（全ステージについて数値シミュレーションで重なりゼロを確認済み）。
また、弾がまれに想定外の方向へ飛んだ場合でもターンが進行不能にならないよう、画面外判定も上下左右すべての方向に対応しました。

## 🚀 セットアップ手順（GitHub Pagesで公開する場合）

### ① GitHubリポジトリを作成

1. [github.com](https://github.com/) にログイン
2. 右上の「+」→「New repository」
3. Repository name を入力（例：`monster-catapult`）
4. **Public** を選択 →「Create repository」

### ② ファイルをアップロード

以下のファイルをリポジトリのルート直下にアップロードしてください（`monster-catapult-v2` フォルダの中身を展開）：

```
index.html
mc-game.js
mc-firebase.js
manifest.json
sw.js
apple-touch-icon.png
icon-58.png 〜 icon-1024.png（フォルダを作らず、そのままルート直下にアップロード）
bg-meadow.jpg / bg-forest.jpg / bg-cave.jpg / bg-snow.jpg（ステージ背景。同じくルート直下）
```

> ⚠️ アイコン画像は `icons/` のようなフォルダを作らず、他のファイルと同じ階層（ルート直下）にそのままアップロードしてください。GitHubのWebアップロード画面はフォルダのドラッグ＆ドロップに失敗しやすいため、フォルダを使わない構成にしています。

### ③ GitHub Pagesを有効化（自動デプロイ）

**方法A：シンプル（追加設定不要、これだけで自動デプロイになります）**
1. リポジトリの「Settings」タブ → 左メニュー「Pages」
2. Source を **「Deploy from a branch」→「main」→「/(root)」** に設定 →「Save」
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセス可能
4. これ以降、**mainブランチにpushするたびに自動的にサイトが更新されます**（追加操作は不要）

**方法B：GitHub Actionsで明示的に自動化する場合**
本フォルダには `.github/workflows/deploy.yml` を同梱しています。これをアップロードすると、push のたびに GitHub Actions がビルド〜デプロイを自動実行し、Service Worker のキャッシュ名も毎回自動更新されるため「更新が反映されない」問題が起きにくくなります。
1. `.github` フォルダごとリポジトリにアップロード（`.github/workflows/deploy.yml` という構造を保ったまま）
2. リポジトリの「Settings」タブ → 左メニュー「Pages」
3. Source を **「GitHub Actions」** に変更
4. 以降、push すると「Actions」タブでデプロイの進行状況が確認できます

方法Aと方法Bはどちらか一方を選べば十分です（併用不可）。

### ④ Firebaseランキング設定（任意）

`mc-firebase.js` 冒頭の `firebaseConfig` にプロジェクト情報を入力すると、オンラインランキングが自動的に有効化されます。未設定でもゲーム自体は問題なく遊べます。

## 📁 ファイル構成

```
monster-catapult-v2/
├── .github/workflows/deploy.yml  # GitHub Actions自動デプロイ設定（任意）
├── index.html       # 画面構造（マップ/モンスター/ショップ/ミッション/ランキング/ゲーム）
```
